# PocketBook Simulator

A browser-based simulator for [PocketBook](https://pocketbook.ch) e-reader applications. Write or port real PocketBook C/C++ apps using the InkView SDK, compile them to WebAssembly with Emscripten, and run them directly in the browser тАФ no device needed.

## How it works

PocketBook apps are written against the **InkView API** (`inkview.h`), which provides drawing primitives, font handling, event dispatching, timers, and UI widgets. This simulator has three layers:

```
PocketBook app (C/C++)
        тФВ  calls
        тЦ╝
src/inkview.c              тАФ thin C shim, bridges each InkView function to JS via EM_JS macros
        тФВ  calls
        тЦ╝
simulator/inkview-emu.mjs  тАФ JavaScript implementation of the InkView API using Canvas 2D
```

The C/C++ source is compiled to WebAssembly by [Emscripten](https://emscripten.org/). The resulting `.mjs` + `.wasm` files are loaded in the browser, where the JS simulator handles all drawing, input, and events.

## Requirements

- [Emscripten SDK (emsdk)](https://emscripten.org/docs/getting_started/downloads.html)
- A local HTTP server (e.g. `python3 -m http.server 8080`)

Install emsdk:

```sh
git clone https://github.com/emscripten-core/emsdk.git ~/emsdk
cd ~/emsdk
./emsdk install latest
./emsdk activate latest
source ~/emsdk/emsdk_env.sh
```

## Building

```sh
# Build the shared InkView object (required before any app)
make lib/inkview.o

# Build individual apps
make demo01
make calc
make touch
make helloworld
make game2048

# Rebuild inkview.o after changing src/inkview.c
rm lib/inkview.o && make lib/inkview.o
```

## Running

Serve the project root with any HTTP server:

```sh
python3 -m http.server 8080
```

Open `http://localhost:8080` тАФ you will see the app launcher. Click an app to run it. The URL hash reflects the active app (`#calc`, `#helloworld`, etc.), so you can bookmark or link directly to an app. The **Home** button (or Escape key) exits back to the launcher.

## Adding a new project

1. Create `projects/<name>/` and put your source files there.

2. Add a Makefile target:

```makefile
myapp: projects/myapp/myapp.c lib/inkview.o
	$(CC) $(CFLAGS) projects/myapp/myapp.c lib/inkview.o \
		-o projects/$@/index.mjs \
		$(EXPORTED_FUNCTIONS) $(EXPORTED_RUNTIME) $(ADDITIONAL_LIBS) $(MODULARIZE)
```

For C++ projects with multiple files, list all sources and add `-I` flags for subdirectory headers.

3. Register the app in `index.html` тАФ add it to `PROJECTS`, the sidebar `<ul>`, and the launcher grid.

4. `make myapp` and reload the browser.

## InkView API coverage

| Category | Functions |
|---|---|
| Screen | `OpenScreen`, `ScreenWidth`, `ScreenHeight`, `ClearScreen` |
| Drawing | `DrawPixel`, `DrawLine`, `DrawRect`, `FillArea`, `FillAreaRect`, `DrawCircle`, `InvertAreaBW` |
| Fonts | `OpenFont`, `CloseFont`, `SetFont`, `StringWidth`, `CharWidth` |
| Text | `DrawString`, `DrawTextRect`, `DrawTextRect2` |
| Events | `EVT_INIT`, `EVT_SHOW`, `EVT_EXIT`, `EVT_POINTERDOWN/UP/MOVE`, `EVT_KEYPRESS`, `EVT_ORIENTATION` тАФ browser arrow/Enter keys mapped to InkView `KEY_*` constants |
| Timers | `SetHardTimer` |
| UI | `OpenMenu`, `DialogSynchro` |
| Orientation | `SetOrientation`, `GetOrientation` тАФ 0┬░/90┬░/180┬░/270┬░ |
| Updates | `FullUpdate`, `PartialUpdate`, `PartialUpdateBW` |
| Misc | `GetDeviceModel`, `GetLangText`, `PanelHeight`, `DrawPanel`, `CloseApp`, `IsInRect`, `iRect` |

## Screen resolution

The simulated device is 600├Ч800 pixels (portrait), matching a PocketBook 6" screen. The Rotate button cycles through all four orientations.
